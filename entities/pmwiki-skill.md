# pmwiki-skill — 项目 Wiki 文档管理 Agent 技能

面向 AI 编程 Agent 的项目 Wiki 文档系统，覆盖从零搭建、分层模板、路线图规划到评估改进的完整生命周期。

- GitHub: [github.com/etrobot/pmwiki-skill](https://github.com/etrobot/pmwiki-skill)
- Stars: 1
- 语言: Python
- 创建: 2026-04-18

## 核心定位

> 这是关于**写文档**的技能，不是开发项目本身。

帮助 Agent 构建、组织和维护项目 Wiki 文档。核心工作流：**评估 → 改进**。

## 两种 Wiki 模式

| 维度 | Lightstart（轻启动） | Precise（精确模式） |
|------|---------------------|-------------------|
| 适用 | 新项目，快速启动 | 重构项目，迭代阶段，多模块 |
| 结构 | 扁平，5 个 .md 文件搞定 | 按层分目录，按主题拆文件 |
| 原则 | 一个文件覆盖全部，够用就好 | 按主题拆分，各归其位 |
| 演进 | 内容膨胀时升级为 Precise | 按需增删子文件，结构已就位 |

### Lightstart 目录结构

```
docs/
├── README.md            # 门面：项目概览、导航
├── 01-architecture.md   # 技术栈、规范、部署
├── 02-data.md           # 表结构、ER 图、缓存
├── 03-business.md       # 核心逻辑、流程图、API
├── 04-interaction.md    # 前端逻辑、UI 设计
├── 05-qa.md             # 测试用例、事故复盘
└── assets/              # 唯一文件夹：图片、截图
```

### 五层文档体系

| 层 | 回答的问题 | Lightstart 写什么 |
|----|-----------|------------------|
| 架构 | 系统怎么建的，为什么 | 技术栈 + 编码规范 + 基础设施 + 架构决策 |
| 数据 | 数据长什么样，存在哪，怎么流 | 表结构 + 数据字典 + 缓存 + 迁移 + 安全 |
| 业务 | 系统做什么，规则是什么 | 核心流程 + 业务规则 + API 接口 + 模块设计 |
| 交互 | 用户看到什么，怎么操作 | UI 规范 + 核心组件 + 用户路径 |
| QA | 怎么验证，出了什么问题 | 测试用例 + 自动化 + 事故复盘 |

## 路线图模板

三种产品类型各有专属路线图模板：

| 类型 | 示例 | 关注点 |
|------|------|--------|
| 本地客户端/桌面应用 | 代码审查工具 | 跨平台、自动更新、插件生态 |
| toC 消费端 | AI 写作工具 | 用户增长、留存、病毒传播、变现时间线 |
| toB SaaS | 项目管理 | 多租户、SSO、合规、私有化部署 |

每个模板遵循四阶段框架 + 技术债务追踪 + 阶段转换标准。

## Wiki 评分系统

### 五个评分维度（等权）

| 类别 | 评分维度 |
|------|---------|
| 项目资产 | 验收标准清晰度 / 责任分配 / 信息时效性 |
| 流程规则 | 步骤可执行性 / 异常分支覆盖 / 与其他流程冲突 |
| 项目语义 | 定义清晰度 / 团队对齐 / 最后确认时间 |
| 经验积累 | 上下文完整性 / 可追溯性 / 可移植性 |
| 操作规程 | 步骤完整性 / 异常分支覆盖 / 最后执行验证 |

综合评分 = (维度分总和 / 维度数) × 20，以叶子条目（具体 .md 文件）为最小单位。

### 优先级规则

| 优先级 | 触发条件 |
|--------|---------|
| P0 紧急 | 综合分 ≤ 30，或任一维度得 1 分，或里程碑 3 天内 |
| P1 高 | 综合分 31-50，或未关闭决策 > 2 天 |
| P2 中 | 综合分 51-70 |
| P3 低 | 综合分 > 70 |

### 评分工作流

1. LLM 逐条评分（1-5 分）→ 输出 JSON
2. Python 脚本 `scorer.py` 计算综合分和优先级
3. 批量评分 + 里程碑紧急模式支持

## 运维能力引入时机

SKILL_ops.md 提供决策矩阵，指导何时引入：
- 定时任务（cron jobs）
- 日志系统
- 告警机制

按产品类型、用户规模、成本控制、团队能力综合判断。

## 与 [[llm-wiki]] 的对比

| 维度 | pmwiki-skill | [[llm-wiki]] |
|------|-------------|------|
| 定位 | 项目文档 Wiki | 知识库 Wiki |
| 内容 | 架构/数据/业务/交互/QA | AI/ML 研究、实体、概念 |
| 模式 | Lightstart / Precise | 三层结构（Raw/Entities/Schema） |
| 评分 | 五维度 LLM 评分系统 | Lint 健康检查 |
| 路线图 | 三种产品类型模板 | 无 |
| 适合场景 | 软件项目文档管理 | 个人知识积累 |

## 适用场景

- 新项目从零搭建文档体系
- 已有项目文档需要规范化
- 需要产品路线图模板（本地/toC/toB）
- 定期评估和改进文档质量
- Agent 辅助维护项目文档

## 局限性

- 仅 1 star，社区尚未验证
- 英文为主，中文支持有限
- 评分系统依赖 LLM，结果可能不稳定
- 与具体开发框架无关，需自行适配
