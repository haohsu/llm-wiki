# IP Character Designer（自媒体 IP 配图生成器）

- **GitHub**: https://github.com/Beatatata/ip-character-designer
- **Stars**: 7
- **作者**: Beatatata（塔塔）
- **语言**: 无代码（纯 Skill/Prompt）
- **创建时间**: 2026-04-15
- **许可**: MIT
- **定位**: 从一个模糊角色概念出发，生成完整的 IP 角色设计全案——角色设定、视觉风格、表情系统、品牌物料、生图提示词

> 「一句话描述 → 角色主视觉 → 场景延展 → 表情包 → 品牌全案」

## 核心能力

| 能力 | 说明 |
|------|------|
| 10 种画风 | 软萌3D / Q版卡通 / 日系漫画 / 扁平插画 / 像素风 / 法式矢量 / 厚涂涂鸦 / 粘土陶瓷 / 写实拟人 / 宝可梦卡牌 |
| 双版本输出 | 白底抠图版（封面/贴纸）+ 场景背景版（图文/传播） |
| 表情包系统 | 9 张基础款 + 扩展表情库（工作/社交/节日系列） |
| 品牌全案 | 配色/Logo/物料/周边/官网全套提示词 |
| 多角色同框 | 支持多张参考图同时生成 |
| 中文提示词 | 全中文，不混英文，复制即用 |

## 塔塔默认模式（核心公式）

从实战中提炼的出图公式，不指定风格时自动套用：

- `3D粘土渲染，柔软的哑光饰面` — 风格基底
- `超广角俯拍视角，夸张的透视比例关系` — 构图标配
- `有冲击力的构图，富有感染力` — 情绪驱动
- `高级质感，极简构图，元素不要杂乱` — 少即是多
- `角色Q版比例，角色要完整不要裁切` — 硬规则
- `玩具般的塑料质感，圆角，温暖柔和的光线` — 质感收尾

## 设计流程

```
Step 0: 环境检测（有CLI？有参考图？）
Step 1: 确认需求（快速生图 / 品牌全案）
Step 2: 选风格（10种，默认3D粘土）
Step 3: 角色设定表（配色/比例/服装）
Step 4: 表情包提示词（9-24张）
Step 5: 品牌物料提示词（Logo/周边/官网）
Step 6: 调用生图（即梦CLI / ListenHub / 纯提示词）
```

## 提示词组装引擎

一条完整提示词 = 6 个模块拼接：

1. **M1 角色描述**：风格 + 物种/性别 + 外貌特征
2. **M2 动作情绪**：具体动作 + 面部表情 + 手势
3. **M3 服装配件**：主要服装 + 品牌标识 + 配件
4. **M4 背景环境**：场景 + 光线 + 氛围
5. **M5 镜头构图**：全身/半身 + 视角
6. **M6 质感风格**：材质 + 色调 + 分辨率

**参考图规则**：有参考图时不描述外貌（交给参考图），无参考图时完整描述。

## 依赖（可选）

- **即梦 CLI**：`dreamina` 命令行工具，有则直接生图
- **即梦网页版**：jimeng.jianying.com（免费注册）
- **ListenHub**：备用生图工具
- 无 CLI 也能用——直接输出提示词，用户去网页版粘贴

## 与生态的关系

- 属于 [[claude-code-skills]] / Cola Skill 生态中的创意设计类技能
- 与 [[huashu-design]] 互补：huashu-design 做 HTML 原型/动画，ip-character-designer 做 IP 角色生图提示词
- 与 [[baoyu-comic]] 方向相近：baoyu-comic 做知识漫画分镜，ip-character-designer 做品牌 IP 角色设计
- 提示词依赖即梦（Dreamina）/ Lovart 等 AI 生图工具，属于 [[ai-image-generation]] 生态的下游应用

## 局限性

- 7 stars，社区规模小
- 纯 Prompt 技能，不含代码/CLI 工具
- 生图质量依赖外部工具（即梦/Lovart/Midjourney）
- 角色一致性受限于 AI 生图模型的 reference 能力
- 无自动验证/质量检查流程
